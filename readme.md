<p>This is a "starter kit" for developing CEP panels.
It relies on <code>cep-bundler</code> to do the heavy lifting.</p>
<p>The <code>cep-bundler</code> does a bunch of things to make your life easier:</p>
<ul>
<li>Compile your <a href="http://www.typescriptlang.org/" rel="nofollow">TypeScript</a> code into a single <code>.js</code> file</li>
<li>Create the necessary xml files (<a href="https://github.com/Adobe-CEP/CEP-Resources/blob/master/CEP_8.x/Documentation/CEP%208.0%20HTML%20Extension%20Cookbook.md#extension-manifest">CSXS/manifest.xml</a>) and <a href="https://github.com/Adobe-CEP/CEP-Resources/blob/master/CEP_8.x/Documentation/CEP%208.0%20HTML%20Extension%20Cookbook.md#remote-debugging">.debug</a>) based on your <code>cep-config.js</code></li>
<li>Create a symlink into the extensions folder so you can test out the extension quickly</li>
<li>"Live Reloads" the code whenever you save a file for instant updates while developing</li>
<li>Syncs your <code>node_modules</code> (<strong>not</strong> the <code>devDependencies</code>) into the bundle</li>
<li>Syncs "public" files (images, svg's, templates, effects, whatever you like) into the bundle</li>
</ul>
<p>For a more detailed explaination, see <a href="https://github.com/mohsinxx365/cep-starter#under-the-hood">under the hood</a></p>

<div class="highlight highlight-source-shell"><pre>npm start</pre></div>
<p>Wait for the packager to have started, then open the program you are targetting.
You can find the extension under <code>Window -&gt; Extensions -&gt; CEP Starter</code>.
Once opened, look for the debug port (or set it yourself) in the <code>cep-config.js</code> and visit <code>http://localhost:DEBUG_PORT_HERE</code> in a Chrome browser (it doesn't work in Firefox).
Now click on the blue link and you will be taken to the debugger.
If you are using version CC2017 or older, go to the settings page (little cog icon on the top right) and disable JavaScript sourcemaps, if you don't do this the debugger will crash, this is <em>not</em> necessary for CC2018.</p>
<p>Happy coding!</p>

<p>You can put any kind of file in the <code>assets</code> folder and it will be put into the <code>build</code> folder.
This way, when <code>browserify</code> is putting your bundle together, you have access to the files in there.
Don't use the <code>import ... from ...</code> syntax, TypeScript will trip over this (only supports loading .ts or .js files)
However, you can still use the require syntax, as shown below.
Just make sure you have a browserify transform configured that transform (in this example) the .scss into a css string.</p>
